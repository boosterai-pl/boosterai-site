# AGENTS.md

## Commands

```bash
# Dev (always source nvm first — non-standard path on this machine)
export NVM_DIR="$HOME/.config/nvm" && [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

docker compose up -d        # start local Postgres (required before pnpm dev)
pnpm dev                    # Next.js + Payload on localhost:3000
pnpm devsafe                # clears .next/ first — use when hot reload breaks

pnpm lint                   # ESLint (uses cross-env NODE_OPTIONS internally)
pnpm format                 # Prettier write
pnpm format:check           # Prettier check

pnpm payload migrate        # apply pending migrations
pnpm generate:types         # regenerate src/payload-types.ts after collection changes
pnpm generate:importmap     # regenerate src/app/(payload)/admin/importMap.js
```

Docker binary is at `/usr/local/bin/docker` — not always in PATH in subshells.

## Environment

Required `.env.local` (copy from `.env.example`):

- `DATABASE_URI` — local: `postgresql://postgres:postgres@localhost:5432/boosterai`
- `PAYLOAD_SECRET` — generate: `openssl rand -hex 32`
- `NEXT_PUBLIC_SERVER_URL` — `http://localhost:3000`
- `BLOB_READ_WRITE_TOKEN` — production only (Vercel Blob), not needed locally

## Architecture

Payload CMS is **embedded inside Next.js** — not a separate service. One `pnpm dev` runs both.

```
src/
├── app/(frontend)/     # public pages — add new pages here
├── app/(payload)/      # Payload admin + REST/GraphQL API — do not modify manually
├── collections/        # Pages.ts, Media.ts, Users.ts — schema definitions
├── migrations/         # auto-generated by Payload — never edit manually
├── payload.config.ts   # add collections/plugins here
└── payload-types.ts    # auto-generated — never edit manually
```

## Payload-specific rules

- **After any collection field change** → run `pnpm payload migrate` + `pnpm generate:types`
- **`src/payload-types.ts`** is auto-generated — never edit it directly
- **`src/migrations/`** files are auto-generated — never edit them manually
- `DATABASE_URI` not `DATABASE_URL` — the scaffold default uses `DATABASE_URL`, this project uses `DATABASE_URI`
- Vercel Blob storage activates automatically when `BLOB_READ_WRITE_TOKEN` is set — no code change needed for prod

## Plugins active

- `@payloadcms/plugin-form-builder` — forms collection in admin, `payment: false`
- `@payloadcms/storage-vercel-blob` — media storage on prod (conditional on env var)

## Tailwind

Uses **Tailwind CSS v4** — config is CSS-based, not `tailwind.config.ts`:

- Import in CSS: `@import "tailwindcss"` (not `@tailwind base/components/utilities`)
- PostCSS config: `postcss.config.mjs` with `@tailwindcss/postcss`

## Reset local database

```bash
docker compose down -v && docker compose up -d
pnpm payload migrate
```
