Header & Footer Architecture

Should the header and footer be global components or managed per-page?

Since you'll have multiple pages (Homepage, Use Cases, Blog, etc.), we need to decide how the header and footer should work.

A

Global Components (Recommended)

Header and Footer as shared React components

// Layout structure:
src/components/
  Header.tsx
  Footer.tsx
src/app/(frontend)/
  layout.tsx → includes Header + Footer
  page.tsx → homepage content only
  use-cases/page.tsx
  blog/page.tsx

Pros

  • Consistent across all pages automatically
  • Single source of truth for navigation
  • Easy to update (change once, affects all pages)
  • Better for SEO consistency
  • Standard Next.js pattern

Cons

  • Less flexibility per-page (though rarely needed)
Content management: Navigation links can be hardcoded in components OR managed via Payload global if you need non-technical editors to change them.
B

Payload-Managed Header/Footer

Header and Footer content stored in Payload CMS globals

// Structure:
src/globals/
  Header.ts → logo, nav items, CTA
  Footer.ts → columns, links, social
src/components/
  Header.tsx → fetches from Payload
  Footer.tsx → fetches from Payload

Pros

  • Non-technical users can update navigation
  • All content in one system (Payload)
  • Can preview changes before publishing
  • Useful if nav items change frequently

Cons

  • More complex setup
  • Additional database queries per page load
  • Overkill if nav is stable
  • More migrations when changing structure
C

Hybrid Approach

Component-based structure + Payload for dynamic parts

// Structure:
src/components/
  Header.tsx → structure hardcoded
  Footer.tsx → structure hardcoded
Payload CTA text in Homepage global
→ "Darmowa konsultacja" comes from CMS

Pros

  • Fast (minimal CMS queries)
  • Structure is stable (developer controlled)
  • Dynamic parts (CTA text) editable by clients
  • Best of both worlds

Cons

  • Split between code and CMS
  • Need developer for nav structure changes

My Recommendation: Option A (Global Components)

For a marketing site where the navigation structure rarely changes, global React components are the simplest and fastest approach. You can always migrate to Payload-managed later if needed. The navigation items (Booster, Usługi, Use cases, Kontakt, Darmowa konsultacja) can be hardcoded since they're unlikely to change frequently.
