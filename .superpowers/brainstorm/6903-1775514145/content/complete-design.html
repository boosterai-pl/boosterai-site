3. Responsive Design Implementation

Breakpoint Strategy

Breakpoint Layout Key Changes
mobile (< 768px) Single column Hamburger menu, stacked cards, larger touch targets (min 44×44px)
tablet (768px+) 2-column grid Service cards in 2 columns, horizontal nav
desktop (1024px+) 3-4 column grid Full horizontal layout, optimized spacing
xl (1280px+) Max-width container Centered layout with max 1280px width

4. Accessibility Standards (WCAG 2.1 AA)

Implementation Checklist

  • ✅ Semantic HTML: <header>, <nav>, <main>, <section>, <footer>
  • ✅ Keyboard Navigation: All interactive elements accessible via Tab/Enter/Space
  • ✅ Focus Indicators: Visible focus rings (custom styled, not removed)
  • ✅ ARIA Labels: aria-label on icon buttons, aria-current on active nav items
  • ✅ Color Contrast: Minimum 4.5:1 for text, 3:1 for large text/UI components
  • ✅ Alt Text: Descriptive alt text for all images (pulled from Payload Media)
  • ✅ Skip Links: "Skip to main content" link at top of page
  • ✅ Form Labels: Proper <label> associations (for contact forms)

5. Performance Optimization

Core Web Vitals Targets

Metric Target Implementation
LCP (Largest Contentful Paint) < 2.5s Next.js Image with priority flag on hero image
FID (First Input Delay) < 100ms Minimal JS, defer non-critical scripts
CLS (Cumulative Layout Shift) < 0.1 Explicit width/height on images, font-display: swap

Optimization Techniques

  • ✅ Image Optimization: Next.js <Image> component with automatic WebP/AVIF conversion
  • ✅ Lazy Loading: Below-the-fold images lazy-loaded automatically
  • ✅ Font Optimization: next/font/google with font subsetting
  • ✅ Static Generation: Homepage as SSG with revalidation (ISR) every 60 seconds
  • ✅ Payload Query Optimization: Select only needed fields, no unnecessary relations

6. Section-by-Section Breakdown

Header Component

  • Layout: Fixed position with backdrop blur on scroll
  • Logo: SVG from /public/, links to homepage
  • Navigation: Booster, Usługi (dropdown?), Use cases, Kontakt
  • CTA: "Darmowa konsultacja" button (prominent, high contrast)
  • Mobile: Hamburger menu with slide-in drawer

Hero Section

  • Layout: Two-column (text left, illustration right) on desktop, stacked on mobile
  • Headline: Large, bold, gradient text effect
  • Subheadline: Secondary text below headline
  • Badge: Small pill with icon showing "300+ automatyzacji"
  • CTA: Primary button → "Kontakt"
  • Illustration: Right side, optimized with Next/Image priority

Sales Booster & AI Booster Sections

  • Reusable Component: <ServicesSection /> accepts props
  • Layout: Section title + grid of service cards
  • Card Grid: 1 col mobile, 2 col tablet, 4 col desktop
  • Card Content: Icon (from Payload Media), title, description
  • Styling: Light background with subtle border, hover lift effect

Współpraca Banner

  • Layout: Full-width blue section with centered text
  • Content: Title with link/CTA

Process Steps Section

  • Layout: 3 large cards with big numbers (01, 02, 03)
  • Card Grid: 1 col mobile, 3 col desktop
  • Styling: Light gradient background, large number typography

Use Cases Preview

  • Layout: Section title + 2 featured case study cards
  • Data Source: Relationship to UseCases collection (max 2)
  • Card Content: Image, category badge, title, excerpt

Footer Component

  • Layout: Multi-column grid (4 col desktop, 2 col tablet, 1 col mobile)
  • Columns: Usługi, Materiały, Kontakt, (optional: About/Social)
  • Bottom: Copyright + privacy links

7. Data Flow

// src/app/(frontend)/page.tsx
export default async function HomePage() {
  const payload = await getPayload({ config })
  
  const homepage = await payload.findGlobal({
    slug: 'homepage',
    depth: 2, // Include media relations
  })

  return (
    <>
      <HeroSection data={homepage.hero} />
      <ServicesSection data={homepage.salesBooster} />
      <ServicesSection data={homepage.aiBooster} />
      <ProcessSection data={homepage.processSteps} />
      <UseCasesPreview data={homepage.useCasesSection} />
      <CTASection data={homepage.cta} />
    </>
  )
}
